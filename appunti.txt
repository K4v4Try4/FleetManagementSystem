docker:
	- docker-compose up: Legge il file YAML e avvia tutti i servizi definiti
	- docker-compose up --build: Forza Docker a ricostruire l'immagine del tuo microservizio partendo dal Dockerfile. È fondamentale usarlo ogni volta che modifichi il codice C#
	- docker-compose up -d: Avvia i container in "background". Questo ti permette di continuare a usare il terminale mentre i servizi girano silenziosamente
	- docker-compose down: Spegne tutti i container e rimuove la rete virtuale creata
	- docker-compose down -v: Cancella i volumi (i dati salvati)
	- docker ps: Ti mostra la lista di tutti i container attivi, il loro ID, l'immagine che usano e, soprattutto, lo Status
	- docker-compose stop: spegnimento
	- docker-compose up -d --build: avvio "migliore"
	- Il "Fantasma" della Cache di Docker: 
		- A volte Docker usa versioni vecchie delle immagini se non "sente" cambiamenti drastici
		- pulizia profonda:
			- docker-compose down
			- docker-compose up -d --build --force-recreate
	- strettamente consigliato:
		- docker compose down -v
		- docker compose up --build --force-recreate
